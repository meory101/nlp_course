WHAT WE DID:
- Stepped back from papers and built up, from scratch, a deep + plain
  understanding of HOW an LLM agent actually works — tokens up to a
  full bug-fix loop. Full explainer saved separately:
     -> ai\how_agents_work_explained.md   (READ THIS to refresh)

THE 4 TRUTHS (the whole machine):
  1. The LLM only WRITES TEXT. "Thought"/"Action" = highest-probability
     next words (tokens -> probabilities -> pick -> repeat).
  2. "Your code" (the program around the LLM, e.g. Claude Code itself)
     is the HANDS — it reads the text and actually runs Read/Edit/Bash.
  3. The TRANSCRIPT is the only memory (LLM forgets between steps;
     re-send the whole transcript every step).
  4. The OBSERVATION is the eyes — paste each result back so the agent
     can see what happened and adjust. This feedback = looks intelligent.

THE LOOP (1 pass): SEND transcript -> LLM writes Thought+Action ->
  PARSE action -> DO it (real tool) -> OBSERVE (paste result back) ->
  REPEAT until finish().

WHY IT "KNOWS" TO OPEN THE FILE: a hidden system prompt ("read before
  editing" + tool list) + training habits make "Read(file)" the
  highest-probability first action. Changing the system prompt changes
  the behavior -> THIS is the API-only experiment lever.

WHAT I BUILD PER IDEA (see how_agents_work_explained.md for detail):
  PART 0 (shared, build once): the loop + an environment + logging
    every step + a budget kill-switch. ~60% of all 3 ideas.
  #16 MEASURE adds: a "judge" (label each step right/wrong) + run many
    task lengths + plot error-rate vs step-number (straight=linear,
    bending up=exponential). Lowest risk, can't fail.
  #18 DETECT adds: a self-monitor prompt each step ("still on task?
    confidence 0-1") + mark the TRUE failure step + check if the alarm
    fires BEFORE failure. One extra LLM call/step.
  #4  REPAIR adds: a critique+redo step + a VERIFIER TOGGLE (feedback
    ON=show test/error vs OFF=hide it) + compare recovery rates +
    over-correction rate. A code agent gives a FREE verifier: run_tests().

AGREED PLAN: build PART 0 + #16 first (insurance + shared data), then
  go deep on ONE of #18 or #4 (pick after reading the 3 anchors).
